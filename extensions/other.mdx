This page covers PHP extensions that do not fit neatly into the caching, database, serialization, networking, imaging, or framework categories. Each entry includes a brief description, the supported PHP version range, and the install command pattern. Replace the version suffix (for example, `@8.4`) with your target PHP version.

<Info>
  All extensions require the tap to be added first. Run `brew tap shivammathur/extensions` once before installing any extension listed here.
</Info>

## Version-limited extensions

Several extensions have an upper PHP version boundary that is lower than the current PHP release. Take note of these before installing:

<AccordionGroup>
  <Accordion title="lua — PHP 5.6 to 7.4 only">
    The lua extension has not been updated for PHP 8.x. If you need Lua scripting in PHP 8+, consider running Lua as a subprocess or via a different integration.
  </Accordion>
  <Accordion title="propro — PHP 5.6 to 7.4 only">
    propro is a dependency of older `pecl_http` releases. On PHP 8.x, `pecl_http` no longer requires propro.
  </Accordion>
  <Accordion title="snmp — PHP 5.6 to 8.3 only">
    The snmp extension formula is not yet available for PHP 8.4 and later in this tap. Use `php@8.3` or earlier for SNMP functionality.
  </Accordion>
  <Accordion title="vld — PHP 5.6 to 8.3 only">
    The vld extension formula is not yet available for PHP 8.4 and later. Use `php@8.3` or earlier for opcode inspection.
  </Accordion>
</AccordionGroup>

<Warning>
  Always run `brew update` before installing an extension to ensure your local formula cache is current. Installing from an outdated cache may pull an older version of the formula.
</Warning>
