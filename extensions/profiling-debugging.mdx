Effective PHP profiling and debugging requires the right extension for the job. The `shivammathur/extensions` tap ships pre-built bottles for the full spectrum of tooling: step-debuggers (Xdebug), coverage drivers (PCOV), hierarchical profilers (XHProf, SPX, Excimer), opcode inspectors (VLD), and production APM agents (New Relic). Each formula is versioned per PHP release so you always get a binary that matches your runtime.

## Prerequisites

Add the tap once and confirm your PHP version is installed before running any install command below:

```zsh
brew update
brew tap shivammathur/extensions
```

## Extensions

<AccordionGroup>

  <Accordion title="xdebug — step debugger and profiler (PHP 5.6–8.6)">
    Xdebug is the most widely used PHP debugging extension. It provides step-through debugging via DAP (Debug Adapter Protocol), call-stack tracing, function-level profiling with cachegrind output, and code-coverage reporting. Xdebug is a **Zend extension** and must be loaded with `zend_extension` in `php.ini`, not the usual `extension` directive.

    **PHP version range:** PHP 5.6 to PHP 8.6

    ```zsh
    brew install shivammathur/extensions/xdebug@8.4
    ```

    Replace `8.4` with the PHP version you are targeting.

    <Warning>
      Xdebug must be configured as a Zend extension. The Homebrew formula writes the correct `.ini` snippet automatically, but if you configure it manually, use:

      ```ini
      zend_extension=xdebug.so
      ```

      Using `extension=xdebug.so` will cause PHP to fail to load the extension.
    </Warning>

    <Tip>
      Xdebug has a significant performance overhead. Use `xdebug.mode=off` in production and only enable the modes you need (e.g. `debug`, `coverage`, `profile`) in development.
    </Tip>
  </Accordion>

  <Accordion title="xdebug2 — Xdebug 2.x legacy (PHP 7.2–7.4)">
    `xdebug2` is the Xdebug 2.x branch, provided for PHP versions that are no longer supported by Xdebug 3. It covers the same step-debugging and profiling feature set as Xdebug 2 and is useful when maintaining legacy applications on PHP 7.2–7.4.

    **PHP version range:** PHP 7.2 to PHP 7.4

    ```zsh
    brew install shivammathur/extensions/xdebug2@7.4
    ```

    Replace `7.4` with `7.2` or `7.3` as appropriate.

    <Warning>
      Like `xdebug`, `xdebug2` is a Zend extension. Load it with `zend_extension=xdebug2.so` in `php.ini`.
    </Warning>

    <Note>
      For PHP 8.0 and later, use the `xdebug` formula (Xdebug 3) instead.
    </Note>
  </Accordion>

  <Accordion title="pcov — code coverage driver">
    PCOV is a self-contained code-coverage driver built specifically for use with PHPUnit and other test frameworks. It is faster and lighter-weight than Xdebug's coverage mode because it does nothing else. Unlike Xdebug, PCOV is a regular extension loaded with `extension`.

    **PHP version range:** PHP 7.1 to PHP 8.6

    ```zsh
    brew install shivammathur/extensions/pcov@8.4
    ```

    Replace `8.4` with the PHP version you are targeting.

    <Tip>
      Do not enable both PCOV and Xdebug at the same time. Both instrument the executor; running them together produces undefined behaviour. Use PCOV for CI coverage runs and Xdebug only for interactive debugging sessions.
    </Tip>
  </Accordion>

  <Accordion title="xhprof — hierarchical PHP profiler">
    XHProf is a function-level hierarchical profiler originally developed at Facebook. It records per-function call counts, wall time, CPU time, and memory usage with low overhead, making it suitable for sampling in development environments. The collected data can be visualised with the XHProf UI.

    **PHP version range:** PHP 7.0 to PHP 8.6

    ```zsh
    brew install shivammathur/extensions/xhprof@8.4
    ```

    Replace `8.4` with the PHP version you are targeting.
  </Accordion>

  <Accordion title="spx — simple PHP profiler">
    SPX (Simple Profiling eXtension) is a low-overhead profiling extension with a built-in web UI. It can capture wall time, CPU time, memory, allocated objects, and more, and presents the results as flame graphs or timeline views without requiring any external tooling.

    **PHP version range:** PHP 5.6 to PHP 8.6

    ```zsh
    brew install shivammathur/extensions/spx@8.4
    ```

    Replace `8.4` with the PHP version you are targeting.
  </Accordion>

  <Accordion title="vld — Vulcan Logic Dumper (opcode inspector)">
    VLD (Vulcan Logic Dumper) dumps the OPcodes generated by the Zend Engine for any PHP script. It is invaluable for understanding how PHP compiles code, diagnosing performance puzzles at the opcode level, or auditing bytecode output.

    **PHP version range:** PHP 5.6 to PHP 8.3

    ```zsh
    brew install shivammathur/extensions/vld@8.3
    ```

    Replace `8.3` with any PHP version from 5.6 to 8.3. VLD is not available for PHP 8.4 or later.

    <Note>
      VLD is not available for PHP 8.4 or PHP 8.5. For opcode inspection on newer PHP versions, consider the `opcache` built-in or other tooling.
    </Note>
  </Accordion>

  <Accordion title="excimer — low-overhead sampling profiler">
    Excimer is a low-overhead sampling (statistical) profiler developed by the Wikimedia Foundation. It interrupts the PHP interpreter at a configurable interval and records a stack sample, producing data suitable for flame graphs. Because it is sampling-based, it has minimal impact on production throughput.

    **PHP version range:** PHP 7.1 to PHP 8.6

    ```zsh
    brew install shivammathur/extensions/excimer@8.4
    ```

    Replace `8.4` with the PHP version you are targeting.
  </Accordion>

  <Accordion title="newrelic — New Relic APM agent">
    The `newrelic` extension is the New Relic application performance monitoring agent for PHP. It instruments web transactions, background jobs, database queries, and external HTTP calls, reporting data to the New Relic platform. A valid New Relic license key and the `newrelic-daemon` process are required at runtime.

    **PHP version range:** PHP 7.4 to PHP 8.4

    ```zsh
    brew install shivammathur/extensions/newrelic@8.4
    ```

    Replace `8.4` with the PHP version you are targeting (7.4–8.4).

    <Warning>
      The New Relic extension is not available for PHP 8.5 or PHP 8.6. Check the [New Relic PHP agent compatibility](https://docs.newrelic.com/docs/apm/agents/php-agent/getting-started/php-agent-compatibility-requirements/) page for the current support matrix.
    </Warning>
  </Accordion>

</AccordionGroup>

## Choosing the right tool

| Goal | Recommended extension |
|---|---|
| Interactive step debugging | `xdebug` |
| CI code-coverage reports | `pcov` |
| Function-level profiling with call graph | `xhprof` |
| Flame graph / timeline profiling with built-in UI | `spx` |
| Low-overhead production sampling | `excimer` |
| Opcode / bytecode inspection | `vld` |
| Production APM and transaction tracing | `newrelic` |

## Upgrading

```zsh
brew upgrade shivammathur/extensions/xdebug@8.4
```

Substitute the formula name and PHP version as needed.
