---
title: "Get started with homebrew-extensions"
description: "Add the shivammathur/extensions tap and install a PHP extension on macOS or Linux in minutes, with pre-built bottles for PHP 5.6 through 8.6."
---

This guide walks you through adding the `shivammathur/extensions` tap to Homebrew and installing a PHP extension. The example below uses Xdebug on PHP 8.4, but the same steps apply to any of the 60+ extensions and any supported PHP version from 5.6 through 8.6.

<Steps>
  <Step title="Update Homebrew">
    Before installing anything, update Homebrew and its formulae to ensure you get the latest bottles:

    ```zsh
    brew update
    ```

    <Tip>Running `brew update` before every install avoids stale formula definitions that can cause build failures or unexpected dependency versions.</Tip>
  </Step>

  <Step title="Check that PHP is installed">
    Verify that the PHP version you need is already installed:

    ```zsh
    php -v
    ```

    If PHP is installed, you will see the version printed to the terminal. If it is not installed, or if you need a specific version that `homebrew/core` no longer ships, install it using the `shivammathur/php` tap:

    ```zsh
    brew tap shivammathur/php
    brew install shivammathur/php/php@8.4
    brew link --force --overwrite shivammathur/php/php@8.4
    ```

    <Note>Extensions from this tap work with PHP installed from either `homebrew/core` or `shivammathur/php`. See [shivammathur/homebrew-php](https://github.com/shivammathur/homebrew-php) for the full list of available PHP versions.</Note>
  </Step>

  <Step title="Add the extensions tap">
    Register the `shivammathur/extensions` tap with Homebrew:

    ```zsh
    brew tap shivammathur/extensions
    ```

    This adds the tap's formulae to your local Homebrew index. You only need to run this once.
  </Step>

  <Step title="Install an extension">
    Install the extension you need by specifying its name and PHP version suffix. For example, to install Xdebug for PHP 8.4:

    ```zsh
    brew install shivammathur/extensions/xdebug@8.4
    ```

    Homebrew downloads a pre-built bottle from the GitHub Container Registry (`ghcr.io`) and writes a configuration file to your PHP `conf.d` directory automatically. No manual `.ini` editing is required.

    To install a different extension or target a different PHP version, replace `xdebug` and `8.4` with the extension name and version you need — for example, `redis@8.3` or `imagick@7.4`.
  </Step>

  <Step title="Verify the extension is loaded">
    Confirm PHP has picked up the new extension:

    ```zsh
    php -m | grep xdebug
    ```

    If the extension is loaded, its name appears in the output. You can also list all loaded extensions at once:

    ```zsh
    php -m
    ```
  </Step>

  <Step title="Restart your web server (if applicable)">
    If you are serving PHP through Apache or Nginx with `php-fpm`, restart the web server to apply the new extension:

    <Tabs>
      <Tab title="Apache">
        ```zsh
        brew services restart httpd
        ```
      </Tab>
      <Tab title="Nginx">
        ```zsh
        brew services restart nginx
        ```
      </Tab>
    </Tabs>

    <Note>This step is only needed if you are running a web server. CLI usage (`php -m`, running scripts) does not require a restart.</Note>
  </Step>
</Steps>

## Upgrading an extension

To upgrade an installed extension to the latest version, use `brew upgrade` with the same formula name:

```zsh
brew upgrade shivammathur/extensions/xdebug@8.4
```

After upgrading, restart your web server if you are running one (see step 6 above).
